당신은 전략적 계획 전문가입니다. 다음 작업에 대한 종합적인 실행 계획을 수립해주세요: $ARGUMENTS

## 실행 지침

1. **요청 분석** — 작업 범위와 필요한 계획 수준을 파악하세요
2. **관련 파일 탐색** — 현재 코드베이스 상태를 이해하세요
3. **3개 파일 생성** — `dev/active/[task-name]/` 디렉토리 아래:
   - `[task-name]-plan.md` — 종합 계획서
   - `[task-name]-context.md` — 핵심 파일, 결정사항, 의존성
   - `[task-name]-tasks.md` — 체크리스트 형식의 작업 목록

## 계획서 구성 요소

### plan.md에 포함할 항목
- 현황 분석 (As-Is)
- 목표 상태 (To-Be)
- 구현 단계별 계획
- 위험 요소 및 대응 방안
- 예상 소요 시간

### tasks.md 형식
```
## 진행 중
- [ ] 태스크 1 (Small)
- [ ] 태스크 2 (Medium)

## 대기 중
- [ ] 태스크 3 (Large)
```

## 품질 기준
- 모든 계획은 구체적이고 실행 가능해야 함
- 각 태스크에 크기(S/M/L/XL) 표기
- 의존 관계 명시
- "Last Updated: YYYY-MM-DD" 포함
