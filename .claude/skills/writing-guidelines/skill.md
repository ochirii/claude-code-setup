# Writing Guidelines Skill

기술 문서, README, 블로그 포스트 등을 작성할 때 자동으로 적용되는 글쓰기 가이드라인입니다.

---

## 문서 작성 원칙

### 구조
- 제목은 H1(#)으로 시작, 섹션은 H2(##), 소섹션은 H3(###) 사용
- 각 섹션 사이에 `---` 구분선 추가
- 목차가 필요한 경우 상단에 배치

### 언어
- 기술 문서: 한국어 + 영어 병기 (예: `에이전트(Agent)`)
- 코드 관련 용어는 영어 원문 유지
- 독자 친화적인 존댓말 사용

### 코드 블록
- 언어 명시 필수: ```python, ```bash, ```json 등
- 실행 가능한 예시 코드 우선
- 주석은 한국어로 작성

### 테이블
- 비교/목록 정보는 테이블로 정리
- 헤더 행 필수 포함
- 정렬 기호(`:-:`, `---:`) 적극 활용

---

## README 작성 템플릿

```markdown
# 프로젝트명

> 한 줄 설명 (한국어)  
> One-line description (English)

## 🚀 시작하기 (Getting Started)
## 📁 구조 (Structure)
## 💡 사용법 (Usage)
## 🤝 기여 (Contributing)
```

---

## 안티패턴 (피할 것)

- 모호한 표현: "좋은", "나쁜", "적절한" → 구체적 수치/기준 제시
- 수동태 남용: "~이 수행됩니다" → "~합니다"
- 불필요한 중복: 같은 내용 반복 금지
- 빈 섹션: 내용 없는 헤더 작성 금지
